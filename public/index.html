Importer & exporter tes données

Récupère tes candidatures depuis un fichier JSON, et exporte-les en Excel, PDF, CSV ou JSON. Tout reste sur ta machine.

📥 Import JSON 📊 Excel 📄 PDF 🧾 CSV 🗂️ Sauvegarde .zip
📥 Importer des candidatures
  1. Onglet Candidatures ou Tableau de bord → ⬆ Importer (JSON)
  2. Choisis un fichier .json ou colle le JSON
  3. Analyser → aperçu (à créer · à compléter · inchangées · ignorées)
  4. Confirmer — rien n'est écrit avant cette étape

La structure JSON

C'est la structure cible : si tu demandes à un outil ou une IA de générer le fichier, donne-lui cette forme. Le niveau racine est un tableau JSON d'objets (ou un objet { "candidatures": [ … ] }). Chaque objet = une candidature ; seuls entreprise et poste sont obligatoires, le reste est optionnel.

Exemple complet avec tous les champs gérés :

candidatures.json
[
  {
    "entreprise": "Studio Lumière",
    "poste": "Chargé de communication",
    "lieu": "Lyon",
    "statut": "Envoyée",
    "date_candidature": "2026-03-12",
    "date_reponse": "2026-03-20",
    "recruteur_nom": "Jordan Doe",
    "recruteur_email": "rh@studio-lumiere.example",
    "lien_offre": "https://studio-lumiere.example/offres/42",
    "salaire": "32 000 €",
    "type_contrat": "CDI",
    "domaine": "Communication",
    "plateforme": "Site de l'entreprise",
    "tags": ["Spontanée", "Priorité haute"],
    "notes": [
      { "label": "Accusé réception", "value": "Candidature reçue le 12/03" }
    ],
    "relances": [
      { "date": "2026-03-19", "canal": "mail", "note": "Relancé, en attente de retour" }
    ],
    "cv_conserve": true,
    "cv_conserve_mois": 24
  }
]

Conservation RGPD · booléen / entier
🔁
Doublons gérés automatiquement — clé entreprise + poste (insensible casse/accents). Une candidature existante est complétée (seuls les champs vides, jamais d'écrasement ; étiquettes fusionnées). Réimporter le même fichier ne crée aucun doublon. L'aperçu signale aussi les quasi-doublons (poste très similaire, ou même entreprise+date) — en alerte seulement, rien n'est fusionné automatiquement.
🛡️
Limites & sécurité — 500 candidatures max par import, fichier ≤ 2 Mo ; notes : titre ≤ 200, contenu ≤ 10 000 ; max 200 notes / 50 étiquettes par candidature. Les URL non http(s) (javascript:, data:…) sont rejetées. Un JSON invalide donne une erreur claire, sans rien écrire.
🔌
Compatibilité (pourquoi des « conversions ») — pour importer aussi des fichiers venus d'ailleurs, l'import tolère en plus : les noms en camelCase (dateCandidature, nomRecruteur…), les dates en JJ/MM/AAAA, les statuts synonymes (Refus → Refusée, Candidature envoyée → Envoyée, inconnu → À postuler) et une date dateRelance seule = une relance effectuée. Si tu maîtrises le fichier, utilise la structure ci-dessus et rien n'est converti.
📤 Exporter & sauvegarder

Onglet Candidatures → bouton « ⬇ Exporter » : choisis le format (Excel / PDF / CSV / JSON) puis des filtres combinables (statut, domaine, plateforme, plage de dates). Un aperçu en direct indique combien de candidatures correspondent avant de télécharger. Le JSON exporté est directement ré-importable.

La Sauvegarde .zip (Paramètres) contient tout — base, fichiers, export JSON et CSV — et n'est jamais filtrée.

Quel format choisir ?

📊
Excel
Analyser, trier, tableaux croisés. Le plus polyvalent en entreprise.
🧾
CSV
Réutiliser ailleurs / réimporter. Interopérable et léger.
🗄️
JSON
Sauvegarde fidèle, structure complète, réimport dans l'app.
📄
PDF
Partager en lecture seule : impression, e-mail.
🗂️
Sauvegarde .zip
Tout conserver : base de données + fichiers importés.

Remplissage auto depuis l'offre —
Dans une candidature, colle le lien d'une offre puis clique sur « ✨ Remplir » : l'app lit les données structurées de la page (gratuit). Pour les sites sans données structurées, active l'extraction par IA (Claude) via une clé API. Le « coller le texte de l'offre » marche aussi sans clé dès que l'IA est activée (recours idéal pour LinkedIn/Indeed).

Pour activer l'IA : crée une clé sur console.anthropic.com, puis lance l'app avec : 

# PowerShell (Windows)
$env:ANTHROPIC_API_KEY="sk-ant-..."
npm start
